study_Day2：双指针与链表题目代码解析
====================================

本文整理以下两个题目：
1. 删除有序数组中的重复项（RemoveDuplicates）
2. 反转链表（ReverseLinkedList）


一、删除有序数组中的重复项
============================

1. 双指针分别做什么
-------------------

slow 指针指向已经完成去重区域的最后一个元素。
fast 指针从左到右扫描数组，寻找与 nums[slow] 不同的新元素。

因为数组已经有序，相同元素一定相邻。当 nums[fast] 与 nums[slow] 不同时，
说明 fast 找到了新的不同元素。此时让 slow 后移一格，再把新元素写入 nums[slow]。


2. 为什么双指针能降低额外空间
-----------------------------

使用新数组或集合保存不重复元素，通常需要 O(n) 的额外空间。双指针直接复用原数组：
fast 负责读取，slow 确定新元素的写入位置。算法只增加两个整数下标，没有创建随
输入规模增长的容器，因此额外空间复杂度为 O(1)。


3. 示例过程
-----------

输入：nums = [1, 1, 2, 2, 3]

开始：slow = 0，nums[slow] = 1
fast = 1：值为 1，重复，不写入。
fast = 2：值为 2，不同；slow 变为 1，写入 nums[1] = 2。
fast = 3：值为 2，重复，不写入。
fast = 4：值为 3，不同；slow 变为 2，写入 nums[2] = 3。

结束后数组前半部分为 [1, 2, 3]，不同元素数量为 slow + 1 = 3。


二、反转链表
============

1. prev、current、next 分别是什么
---------------------------------

prev：指向 current 的前一个节点，也是已经完成反转部分的头节点。初始为 null。
current：指向当前正在处理、即将改变 next 指向的节点。初始为 head。
next：临时保存 current 原来的下一个节点，即尚未处理部分的头节点。


2. 为什么必须先保存 next
-------------------------

链表只能通过节点的 next 找到后续节点。执行 current.next = prev 后，current.next
已经从指向后一个节点改成指向前一个节点。如果没有提前保存原来的 current.next，
程序就失去了通往剩余链表的路径，后续节点将无法继续访问。

每轮循环的顺序必须是：
1. next = current.next：保存后续链表。
2. current.next = prev：反转当前指针。
3. prev = current：让 prev 前进一步。
4. current = next：继续处理原链表的下一个节点。


3. 示例过程
-----------

输入：1 -> 2 -> 3 -> null

初始：prev = null，current = 1。
第一轮：保存 next = 2，将 1 指向 null；prev = 1，current = 2。
第二轮：保存 next = 3，将 2 指向 1；prev = 2，current = 3。
第三轮：保存 next = null，将 3 指向 2；prev = 3，current = null。

循环结束时 prev 指向新头节点 3，结果为 3 -> 2 -> 1 -> null。
