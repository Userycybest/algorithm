study_Day3：二分查找题目代码解析
================================

本次完成三个题目：
1. 标准二分查找（BinarySearch）
2. 搜索插入位置（SearchInsertPosition）
3. 在排序数组中查找元素的第一个和最后一个位置（FindFirstAndLastPosition）

一、二分查找的使用条件
======================

二分查找通常用于已经排序的数据。每次比较中间元素后，可以排除一半不可能包含答案的区间，
所以时间复杂度为 O(log n)。如果数组没有排序，就不能根据中间值判断应该舍弃左半边还是右半边。

二、标准二分查找
================

1. 区间定义
-----------

标准实现使用闭区间 [left, right]：left 和 right 指向的位置都可能是答案。
初始化为 left = 0，right = nums.length - 1。

因为区间两端都有效，所以循环条件是 left <= right：
- nums[mid] < target：mid 不可能是答案，left = mid + 1；
- nums[mid] > target：mid 不可能是答案，right = mid - 1；
- nums[mid] == target：直接返回 mid。

2. 为什么这样计算 mid
---------------------

使用：

mid = left + (right - left) / 2

它与 (left + right) / 2 的结果相同，但能避免 left + right 很大时发生整数溢出。

三、搜索插入位置
================

该题不仅要查找相等值，还要在目标不存在时返回插入位置。它可以转化为：

“查找数组中第一个大于或等于 target 的位置。”

实现使用左闭右开区间 [left, right)，初始化 right = nums.length。
nums.length 也是一个合法答案，表示目标值应该插入数组末尾。

循环条件为 left < right：
- nums[mid] < target：mid 以及它左边都不可能是答案，left = mid + 1；
- nums[mid] >= target：mid 可能是答案，保留 mid，right = mid。

循环结束时 left == right，该位置就是第一个大于或等于 target 的位置。

四、查找第一个和最后一个位置
============================

如果找到任意一个 target 后再向两边线性扫描，最坏情况下需要 O(n)，不符合 O(log n) 的目标。
因此需要执行两次边界二分查找：

1. lowerBound：查找第一个大于或等于 target 的位置；
2. upperBound：查找第一个严格大于 target 的位置。

目标值的最后一个位置就是：upperBound(target) - 1。

例如：nums = [5, 7, 7, 8, 8, 10]，target = 8
- lowerBound 返回 3；
- upperBound 返回 5；
- 最后一个 8 的位置为 5 - 1 = 4；
- 最终结果为 [3, 4]。

lowerBound 只返回“应该插入的位置”，并不保证目标值真实存在。因此还需要判断：
- first 是否已经等于数组长度；
- nums[first] 是否等于 target。

如果不满足，返回 [-1, -1]。

五、最容易出现的错误
====================

1. 闭区间写法使用 left < right，导致只剩一个元素时没有检查；
2. 闭区间中更新为 left = mid 或 right = mid，可能造成死循环；
3. 左闭右开写法却把 right 初始化为 nums.length - 1，遗漏末尾插入位置；
4. lowerBound 返回后没有确认目标值是否存在；
5. 找到目标后向左右逐个扫描，使最坏时间复杂度退化为 O(n)；
6. 使用 target + 1 查找右边界时，没有考虑 target 为 Integer.MAX_VALUE 的溢出问题。

六、复杂度总结
==============

三个题目都只使用若干整数变量：
- 时间复杂度：O(log n)；
- 额外空间复杂度：O(1)。

七、今天需要掌握的重点
======================

1. 写二分查找前先确定搜索区间是 [left, right] 还是 [left, right)；
2. 循环条件和左右边界更新必须与区间定义保持一致；
3. 标准二分查找寻找“某个等于目标的位置”；
4. lowerBound 寻找“第一个大于或等于目标的位置”；
5. upperBound 寻找“第一个严格大于目标的位置”；
6. 边界二分查找可以解决插入位置、首次出现和最后出现等问题。
